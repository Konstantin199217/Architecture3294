                                       Команда (Command)
=======================================================================================================================
    Поведенческий паттерн проектирования, который превращает запросы в объекты, позволяя передавать их как аргументы
при вызове методов, ставить запросы в очередь, логировать их, а так же поддерживать отмену операций.
=======================================================================================================================
    Стоит использовать, когда:
          - Мы хотим параметризовать объекты выполняемым действием;
          - Требуется определять, ставить в очередь и выполнять запросы в разное время;
          - Надо поддержать отмену операций;
          - Требуется поддержать протоколирование изменений, чтобы выполнить их повторно после аварийной остановки
            системы;
          - Мы хотим структурировать систему на основе высокоуровневых операций, построенных из примитивных;
=======================================================================================================================
                                    Преимущества и недостатки Command
    +
      * Убирает прямую зависимость между объектами вызывающими операции и объектами, которые их выполняют;
      * Позволяет реализовать простую отмену и повтор операций;
      * Позволяет реализовать отложенный запуск операций;
      * Позволяет собирать сложные команды из простых;
      * Реализует принцип открытости - закрытости;
    -
      * Усложняет код программы из-за введения дополнительных подклассов;
=======================================================================================================================
