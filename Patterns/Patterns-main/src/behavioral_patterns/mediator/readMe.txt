                                       Посредник (Mediator)
=======================================================================================================================
    Поведенческий паттерн проектирования, который позволяет уменьшить связанность классов между собой, благодаря
перемещению этих связей в один класс-посредник.
=======================================================================================================================
    Стоит использовать, когда:
          - Есть объекты, связи между которыми сложны и четко определены Получающиеся при этом связи не структурируются
            и трубны для понимания;
          - Нельзя повторно использовать объект, поскольку он обменивается информацией со многими другими объектами;
          - Поведение распределенное между несколькими классами, должно поддаваться настройке без порождения
            множества подклассов;
=======================================================================================================================
                                    Преимущества и недостатки Mediator
    +
      * Устраняет зависимости между компонентами, позволяя повторно их использовать;
      * Упрощает взаимодействие между компонентами;
      * Централизует управление в одном месте;
    -
      * Посредник может сильно увеличить код;
=======================================================================================================================
