                                       Шаблонный метод (Template method)
=======================================================================================================================
    Поведенческий паттерн проектирования, который определяет скелет алгоритма, перекладывая ответственность за некоторые
его шаги на подклассы. Паттерн позволяет подклассам переопределять шаги алгоритма, не меняя его общей структуры.
=======================================================================================================================
    Стоит использовать, когда:
          - Чтобы однократно использовать инвариантные части алгоритма, оставляя реализацию изменяющегося поведения
            на усмотрение подклассов;
          - Когда надо вычленить и локализовать в одном классе поведение, общее для всех подклассов, дабы избежать
            дублирования кода;
          - Для управления расширениями подклассов;
=======================================================================================================================
                                    Преимущества и недостатки Template method
    +
      * Облегчает повторное использование кода;
    -
      * Жесткое ограничение скелетом действующего алгоритма;
      * С ростом количества шагов шаблонный метод становится слишком сложно поддерживать;
=======================================================================================================================
