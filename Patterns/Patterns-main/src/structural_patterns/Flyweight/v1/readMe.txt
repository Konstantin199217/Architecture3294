                                       Приспособленец (Flyweight)
=======================================================================================================================
    Структурный паттерн проектирования, который позволяет вместить больше объектов в отведенную оперативную память.
Это позволяет экономить память, разделяя общее состояние объектов между собой, вместо хранения одинаковых данных
в каждом объекте.
=======================================================================================================================
    Стоит использовать, когда:
          - В приложении используется большое число объектов, из-за этого накладные расходы на хранение велики;
          - Большая часть состояния объектов выносится вовне;
          - Многие группы объектов заменяются относительно небольшим числом разделенных объектов, поскольку внешнее
            состояние вынесено;
          - Приложение не зависит от идентичности объектов;
=======================================================================================================================
                                    Преимущества и недостатки Flyweight
    +
      * Экономит оперативную память;
    -
      * Расходует процессорное время на поиск/вычисление контекста;
      * Усложняет код программы из-за введения множества дополнительных классов;
